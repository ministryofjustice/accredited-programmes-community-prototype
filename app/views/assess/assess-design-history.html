Concept 5 IN PROCESS

Request more information (tasklist)

The programme team will request more information from the probation practitioner if required. The probation practitioner will then recieve a tasklist to complete.

Concept 4 IN PROCESS

Live updates

This journey includes a feature where both the programme team and probation practitioner can update the referral with different forms of additional information.

Concept 3 IN MIRO

Request more information (form)

This journey includes imported data plus an area for the programme team to request more information from the probation practitioner. The first iteration of this is in Miro.

Concept 2 IN PROCESS

Imported data only

This journey only contains information that has been pulled in automatically from other services. We want to test with users if this is enough to make an assessment.

Concept 1 TESTED

Original custody prototype

This journey has been imported from the custody prototype. We have decided to use this as our foundation of research in this project to help us understand what the differences are between custody and community.
